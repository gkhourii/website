# Roundabout Software

A web-based geography encyclopedia. My first personal project and web dev experience.

## What went well
*The text box and user input worked, properly displaying the country and its corresponding facts + flag.

*Pictures were successfully displayed in their proper format, after much troubleshooting

*The back button functioned as intended.

*I gained a good feel for writing basic HTML and PHP.

*I experimented with styling, such as colors, spacing, font size, and picture borders. 

## What went wrong
The original core MVP was an online map quiz, where the user would identify amd select a country by name from a map GUI, similar to what you might find on sporcle.com.
However, given my lack of experience at the time as well as the time constraints of the assignment, I resorted to my backup plan: a web database displaying a handful of facts about each European country.
Also, I initially tried to use JDBC drivers to connect Java code to the MySQL database, which would've allowed me to write API queries in the more familiar Java. For instance, I wanted to treat a table as an array so I could use for loops to fill/modify it. Despite my efforts, however, the connection failed, so I instead inserted them manually into the MySQL tables.

## Areas for Improvement
*The country dropdown menu should auto-fill the text box to the selected country.

*The landing page could benefit from additional information and features

*The bonus page should load in the same tab instead of redirecting to a new tab

## Overall thoughts
Looking back, I'm satisfied with the simplicity of the final product. This was my first full-stack website I had ever created, and my first time using HTML, PHP, and MySQL,
and by the final week of development I had grown much more comfortable with these tools; my only regret is not diving into them sooner. 
I realized that the time I spent fiddling with the JDBC drivers was a way to remain in my comfort zone (i.e., Java) and avoid the discomfort of learning new tools (i.e., HTML, PHP, MySQL). 
Instead, I should’ve immediately started experimenting with HTML and PHP, as I would’ve accumulated enough experience with it to push the project just a bit further.
In this regard, my takeaway for future projects is to embrace new tools, adapt quickly to new programming paradigms, and to plan accordingly to tailor my existing skills to the requirements of the project.
